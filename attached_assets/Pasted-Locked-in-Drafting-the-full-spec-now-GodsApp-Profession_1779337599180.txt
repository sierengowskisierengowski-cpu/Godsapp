What GodsApp Is
GodsApp is a native Linux desktop application that consolidates professional security auditing, network reconnaissance, vulnerability assessment, forensic analysis, threat intelligence, and offensive security tooling into a single integrated environment. It is the desktop equivalent of opening a full security operations room — comparable in scope to Kali Linux's toolset, but unified under one application with shared workspaces, evidence handling, and reporting.
Every tool in GodsApp operates at full capability. No watered-down versions. No simulated results. No mock data. Every scan produces real output. Every analysis is genuine. Same legal standing as Nmap, Wireshark, Metasploit, and other professional security tools — the tools themselves are legal, the user is solely responsible for ensuring authorization to use them against any target.
GodsApp is standalone but Meli-aware: if Meli is detected on the system, GodsApp surfaces additional cross-reference features that query Meli's honeypot data.
Tech Stack
Frontend: Native GTK4 with PyGObject (Python bindings) and libadwaita for modern controls
Backend: Python 3.12
Database: SQLite for local workspace storage with optional PostgreSQL for power users with large datasets
Background workers: Python threading + asyncio for parallel scans, with systemd user services for long-running daemons
Plugin system: Python module loading from ~/.config/godsapp/plugins/
REST API: FastAPI for local programmatic access
CLI mode: Click-based subcommands sharing the same backend
Packaging: PyInstaller for binary, with Arch PKGBUILD, .deb, AppImage, and Flatpak targets

Setup Wizard
After authorization, a multi-step wizard:

Welcome — name, version, author Joseph Sierengowski, brief description
Master password creation — min 14 chars, complexity meter, confirmation
2FA setup (optional) — TOTP via QR or YubiKey enrollment
API key entry (optional) — fields for Shodan, Censys, VirusTotal, AbuseIPDB, GreyNoise, HIBP, MISP, OTX (each skippable)
System dependency check — scans for nmap, nuclei, hydra, etc. Shows which are present, which are missing, with one-click install commands for the user's distribution
Wordlist installation (optional) — offers to download SecLists, rockyou, and other common wordlists with progress bars
Nuclei template install — downloads nuclei community templates
CVE database init — downloads CVE database for offline lookups
First workspace creation — creates "Default Workspace" or prompts for custom name
Meli detection — checks if Meli is installed, displays integration status
Setup complete — summary, "Launch GodsApp" button

Plugin System

Plugin API
pythonfrom godsapp.plugin_api import Plugin, ToolCategory, register

@register
class MyCustomTool(Plugin):
    name = "My Custom Tool"
    category = ToolCategory.NETWORK
    description = "Does something useful"
    author = "Plugin Author"
    version = "1.0.0"
    
    def get_inputs(self):
        return [
            {"name": "target", "type": "text", "required": True},
            {"name": "depth", "type": "int", "default": 3},
        ]
    
    def run(self, inputs, context):
        # context provides: workspace, evidence_locker, logger, api_keys
        # Return findings, raw output, evidence
        result = do_something(inputs["target"])
        context.evidence_locker.capture(result, source="my_custom_tool")
        return {"output": result, "findings": []}
